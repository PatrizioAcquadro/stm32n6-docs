The Software Toolchain

After understanding the hardware, it is time to look at the software side. This chapter walks through every tool involved in the deployment pipeline — what it does, how it fits together, and where to find the detailed code explanation in Part 2 and Part 3.

ST Model Zoo ModelZoo Services ST Edge AI Core STM32CubeIDE
Key Insight — Keep this in mind

Before diving into the tools, there is one idea that ties everything together: the division of labor between Python and C.

🐍 Python — Host side
Handles training, quantization, and evaluation. Runs on your laptop. Produces the optimized and quantized model files (.tflite, .onnx) ready for conversion.
⚡ C — Board side
The C project is what actually runs on the board. ST Edge AI Core generates the C arrays and configuration headers from the Python model, which are then compiled by STM32CubeIDE and flashed via ST-Link.

Chapter Scope

Having examined the hardware platform (the STM32N6570-DK, its Neural-ART NPU and its memory hierarchy) this section transitions to the software toolchain to detail the deployment workflow for neural network firmware.

Each tool is analyzed based on its role, required configuration files, and practical usage. Direct links are provided to the corresponding source code documentation in Part 2 — Code Reference and Part 3 — Module Groups, where individual functions are analyzed line-by-line.

This chapter is structured into three parts, reflecting the logical division of the deployment pipeline:

Part A — Python Pipeline
ST Model Zoo and ModelZoo Services: Covers model sourcing, post-training INT8 quantization, and pipeline orchestration using a centralized YAML configuration file.
Part B — ST Edge AI Core
The Neural-ART Compiler: Translates quantized .tflite/.onnx models into hardware-optimized C arrays. It assigns operational steps (NPU epochs), generates memory pool (.mpool) mappings, and outputs profiling reports to bridge the Python-to-C gap.
Part C — STM32CubeIDE & C Firmware
Compilation and Board Deployment: STM32CubeIDE compiles the generated C project and flashes it via ST-Link. This section details the firmware components and inference loops executing natively on the STM32N6570-DK.

Pipeline Overview

The complete deployment pipeline consists of four distinct software components, each responsible for a specific phase of the workflow. These components execute in a strict sequential order:

The dashed vertical line in the diagram denotes the conceptual boundary between the Machine Learning environment and the Embedded Software environment. Crucially, all four toolchain components execute on the host machine. The Python ecosystem (left) governs model design, training, and INT8 quantization. The C ecosystem (right) handles the hardware-specific translation of the model into C arrays via ST Edge AI Core, followed by cross-compilation within STM32CubeIDE. Only the resulting binary executable is flashed to and executed natively on the STM32N6570-DK hardware.

The proprietary ST compiler facilitating the transition from the Python environment to embedded C.
