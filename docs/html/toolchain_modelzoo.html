3.1 — ST Model Zoo

Where the models come from

The ST Model Zoo is the official, open-source GitHub repository maintained by STMicroelectronics. It provides a curated collection of pre-trained machine learning models, training scripts and evaluation pipelines, all rigorously validated for optimized deployment across the STM32 ecosystem.

🔗 github.com/STMicroelectronics/stm32ai-modelzoo

Purpose and Practical Utility

Training a neural network architecture from scratch necessitates extensive datasets, high-performance GPU (or TPU) clusters and significant computational time. Within the context of embedded machine learning workflows, this approach is frequently inefficient. The ST Model Zoo mitigates this bottleneck by supplying models that are already trained, quantized and validated against reference STM32 hardware. This allows developers to bypass the training phase and transition directly to hardware-specific compilation and deployment.

As of Release 4.0, the repository natively supports three primary deep learning frameworks: TensorFlow, PyTorch and ONNX and provides comprehensive latency, memory footprint and accuracy benchmarks across both Microcontroller (MCU) and Microprocessor (MPU) target platforms.

What’s new in release 4.0 (latest):
  • Comprehensive PyTorch support introduced for Image Classification and Object Detection pipelines.
  • Full integration with the STEdgeAI Core v3.0.0 compiler.
  • Expansion of reference use cases: Face Detection, Arc Fault Detection and Re-Identification.
  • Introduction of mixed-precision quantization models (4-bit Weights, 8-bit Activations).
  • Updated framework compatibility matrices: Keras 3.8.0, TensorFlow 2.18.0, PyTorch 2.7.1 and ONNX 1.16.1.
  • Implementation of a Docker-based environment, providing a ready-to-use containerized software stack.

Architectural Deep-Dive: Pose Estimation

The pose_estimation/ directory encapsulates five distinct model architectures, each optimized for specific landmark detection tasks:

  • movenet — Single-pose detection, outputting 13 or 17 keypoints  Deployed in this project
  • yolov8n_pose — Multi-pose detection, outputting 17 keypoints (COCO standard)  Deployed in this project
  • yolov11n_pose — Multi-pose detection, outputting 17 keypoints (latest YOLO iteration)
  • handlandmarks — Single-pose detection, mapping 21 distinct hand keypoints
  • headlandmarks — Single-pose detection, extracting a dense 468/478 facial keypoint mesh

Repository Initialization and Setup

The repository utilizes Git Large File Storage (Git LFS) for managing substantial binary assets, specifically pre-trained model weights. Git LFS must be installed and initialized prior to cloning; failure to execute this prerequisite will result in the retrieval of unresolved pointer files rather than the actual model binaries.

# 1. Install Git LFS (Ubuntu/Debian environment)
sudo apt-get install git-lfs

# 2. Initialize Git LFS within the local environment
git lfs install
# Expected stdout: Git LFS initialized.

# 3. Clone the repository into the designated workspace
git clone https://github.com/STMicroelectronics/stm32ai-modelzoo.git

# 4. Navigate to the target model directory
cd stm32ai-modelzoo/pose_estimation/movenet/ST_pretrainedmodel_public_dataset/
Critical Workspace Hierarchy: The Model Zoo repository must be cloned as a sibling directory to the stm32ai-modelzoo-services repository. The default configuration templates provided by the Services framework rely on relative paths mapped to this specific structure. Deviating from this hierarchy requires manual, comprehensive reconfiguration of all model_path variables within the localized user_config.yaml file.
