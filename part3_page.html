Group
STM32N6570-DK Firmware Application
Complete firmware for real-time pose estimation on STM32N6
Open full group →

This group contains all C and header files that are compiled into the firmware and flashed onto the STM32N6570-DK board. They implement the complete real-time application: camera acquisition, NPU inference trigger, postprocessing, and display. All files in this group run on the Cortex-M55 CPU at 800 MHz and interact with the Neural-ART NPU via the ll_aton runtime.

app_config.h View →
Role: Generated by gen_h_file.py — the only generated file included by handwritten C. Defines NN dimensions, keypoint count, postprocess type, and confidence threshold. Change the model → regenerate this file → recompile. All other files read from it.
Generated #include by all
main.c View →
Role: Entry point after reset. HAL + clock + peripheral init. Starts DCMIPP. Runs the infinite inference loop: snapshot trigger → LL_ATON_RT_Main() → postprocess → render. The top of the C call graph.
Entry point Inference loop
app_camerapipeline.c / .h
.c → .h →
Role: MIPI CSI-2 + DCMIPP configuration. Dual-pipe: display pipe writes to PSRAM continuously (LCD background), NN pipe delivers a cropped snapshot to npuRAM4 on demand. Zero CPU for memory transfers.
DCMIPP dual-pipe DMA-driven
display_spe.c / .h
.c → .h →
Role: MoveNet postprocessor. Reads float32 heatmaps (48×48×13) from npuRAM5, finds argmax per channel, filters by AI_POSE_PP_CONF_THRESHOLD, draws 13-keypoint skeleton on the LCD foreground layer using display_keypoints_13.h connectivity table.
MoveNet decoder LCD render
display_mpe.c / .h
.c → .h →
Role: YOLOv8n-pose postprocessor. Reads detection output (bounding boxes + 17 keypoints per person), applies NMS filtering, draws bounding boxes and 17-keypoint COCO skeletons for all detected persons simultaneously.
YOLOv8 decoder Multi-person
crop_img.c / .h
.c → .h →
Role: Image crop utilities used by the NN pipe to extract and resize the region of interest from the full camera frame before feeding it to the NPU input buffer.
display_keypoints_13.h / _17.h
13 → 17 →
Role: Skeleton connectivity tables — arrays of index pairs defining which keypoints to connect with lines. 13-keypoint for MoveNet (no head landmarks), 17-keypoint COCO standard for YOLOv8.
main.h  •  utils.h
main.h → utils.h →
main.h: Global typedefs, peripheral handle declarations, extern function prototypes used across all C files.  |  utils.h: Shared utility macros and inline helper functions (timing, debug, buffer alignment).

Python Deployment Pipeline
Python scripts orchestrating the full model-to-firmware pipeline
Open full group →

This group contains all Python files that run on the host PC. They are organised by sub-folder, each responsible for a distinct phase of the pipeline. The files are grouped below by their functional role, from orchestration down to the individual processing steps.

▶ Orchestration layer
stm32ai_main.py View →
Entry point. Hydra @main. Reads YAML, dispatches to operation mode.
parse_config.py View →
Validates every field of user_config.yaml. Fail-fast design.
deploy.py View →
Deployment orchestrator. Calls gen_h_file → common_deploy → CubeIDE.
common_deploy.py View →
Invokes ST Edge AI Core CLI, CubeIDE headless, SigningTool, CubeProgrammer.
🔗 Python → C bridge
gen_h_file.py View →
Reads TFLite tensor shapes → writes app_config.h. The only file where Python directly controls C behaviour.
external_memory_mgt.py View →
Patches the linker script to place weight binary at 0x70380000 in OctoFlash.
models_mgt.py View →
Model loading utilities and ST Edge AI runner interface for on-board evaluation via stedgeai_n6 target.
⚙ Data processing
quantize.py View →
PTQ: float32 → INT8. TFLite path for .h5, ONNX path for .onnx.
preprocess.py View →
Dataset loading + rescaling + single-image preprocessing.
data_loader.py View →
COCO JSON parsing, image loading, keypoint normalisation.
postprocess.py View →
Heatmap argmax (MoveNet), YOLO NMS, hand/head decoders.
evaluate.py View →
Runs inference on test set. Supports host, stedgeai_host, stedgeai_n6.
metrics.py View →
OKS (MoveNet), mAP@[0.5:0.95] (YOLOv8). Full COCO eval protocol.
